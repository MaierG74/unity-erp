Variant 2 · idea 2

Compact readiness rows

Two-line row per component: code + numbers top, description below, swap icon pinned right. Tabular columns line up. Subtle zebra striping plus destructive tint on shortfall rows. Description stays visible so operators can identify components ("Hollow SSS Bar Handle" not just "RIH1516") and the swap icon keeps the existing same-category swap flow reachable without an extra click. Click the row to expand for full breakdown (In stock / Reserved / On order / Global). Reduces ~25 lines → ~10.

Variant 3 · idea 3

One-line cutlist materials

Each board-type group becomes a single line — group label on the left, materials on the right. Edging hides unless overrides exist; a small teal flag signals "click for per-part detail". Saves ~6 lines vertically and reads as a summary, not a data dump.

Variant 4 · combined recommended

Everything collapsed by default + per-row actions

All four sections start collapsed. The status sentence ("1 component short") moves into the panel header so it's always visible. The operator clicks the section they need.

Component readiness header has ＋ Reserve all (existing order-scoped API; free) and a "1 short" pill. Three icons per row:

  • ⟳ swap — always enabled. Phase 1 wiring.
  • ＋ reserve — enabled when Avail > 0; claims on-hand stock. Disabled on the shortfall row because there's nothing to reserve. Needs Phase 3 backend (new reserve_order_component RPC).
  • 🛒 order — enabled when Short > 0; opens the existing Order Components dialog with this row pre-selected and the shortfall pre-filled. No new backend — pipes into the current PO-draft flow.

On the RIH1516 row (shortfall, no stock): reserve is correctly disabled, order is enabled. On the three healthy rows: reserve is enabled, order is disabled.

Long descriptions truncate with hover tooltip.

Click each chevron to expand a section.
