Per-row action semantics (state-aware):
  • ⟳ swap — always enabled. Opens existing same-category swap dialog. (Phase 1 wiring; works today.)
  • ＋ reserve — enabled when Avail > 0. Claims on-hand stock for this line. Disabled on the shortfall row because there's nothing to reserve. Needs Phase 3 backend (new reserve_order_component RPC + API route).
  • 🛒 order — enabled when Short > 0. Opens the existing Order Components dialog with this row pre-selected and the shortfall quantity pre-filled. No new backend — pipes into the current PO-draft flow.
